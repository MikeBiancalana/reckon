# Implementation Plan — reckon-ih5g (v1-T8: note tool `rk note` + linking)

## Summary

T8 delivers the zettelkasten pillar as **thin CLI orchestration over already-shipped generic machinery**. `internal/node` already extracts body `[[wikilinks]]` into `references` edges and ref-valued frontmatter props into typed edges; `internal/index`'s `resolveEdges` already re-resolves every edge's `dst_key` against `nodes.ulid` then `_aliases.alias` on every reconcile pass, so dangling-link auto-resolution, backlink derivation, and unresolved-edge queryability are free consequences of T1/T2 — not code this ticket writes. The default `LogParser`/`MarkdownParser` handles one-file-per-note with zero parser-dispatch work. What T8 actually builds is: (1) a `rk note create` verb that writes `notes/<slug>.md` via the established `NewNode → set fields → Render → Parse → writeFileAtomic` recipe; (2) a `rk note show <ref>` read verb that opens the index, reconciles, and renders a note's fields plus index-derived forward-links and backlinks; (3) a `rk note rename <ref> "New Title"` verb implementing the alias-redirect contract; (4) a `rk note index` porcelain that regenerates `notes/**/index.md`; and (5) the slug/alias machinery that ties it together.

Two findings shape the design. First, **the index populates `_aliases` only from frontmatter `aliases:`, never from a filename** — so `rk note create` must *self-mint its own slug into the note's `aliases:` line*, or `[[slug]]` links from other notes will resolve in Obsidian but never populate `dst_key` in reckon's graph, silently failing the "forward/backlinks queryable" criterion for the primary Obsidian-native link style. This is the load-bearing wiring, decided in `composable-redesign.md` A#6 ("each tool auto-mints a canonical alias"). Second, **the rename-retains-alias contract has no safe primitive for Obsidian block-list-style `aliases:`**: today's `InsertField`/`SetField` silently corrupt such a file (verified in the acceptance-criteria analysis — `InsertField` keys its existence check on `fieldSpans`, but block-list values are recorded in `frontmatter` with no span, so it prepends a duplicate `aliases:` line whose value is dropped on reparse). The plan adds one small, fuzz-gated `internal/node` primitive — `SetAliases([]string)` — that handles all three alias shapes (scalar span, block list, absent) by collapsing to canonical flow form; this is the only node-package change required, everything else is CLI-layer.

Design decisions favor minimalism and existing precedent: slugs are lowercased-Unicode, Obsidian-filename-safe, and validated (empty and reserved-`index` slugs rejected); slug/slug-alias collisions are **rejected at create and rename time via a filesystem scan of `notes/`** (the note-vs-note case that actually matters), while the truly-global cross-type collision surface stays the existing reactive reconcile `alias_collision` warning — honoring "index-enforced uniqueness" as shipped without rewriting index semantics; `index.md` generation is a **`rk note index` porcelain, not folded into reconcile** (writing synced vault files as a side effect of a lazy read-path reconcile would be surprising and would churn git on every `rk … list`/`show`; an explicit verb matches the `rk index`/T4/T5 precedent). New code lands in new `internal/cli` files that attach subcommands to the existing legacy `noteCmd`, leaving the legacy DB-backed `note`/`notes` surface untouched pending T9. No `os.Exit`, wrapped errors, `requiresDB:"false"`, `output.Writer`, `--quiet` suppression, and `resolveAuthor`/`writeFileAtomic` reuse throughout.

## Design decisions

### Slug algorithm + Obsidian compatibility
`slugify`: trim → lowercase → keep `unicode.IsLetter`/`IsDigit`/`-`/`_`, map runs of whitespace to a single `-`, drop everything else (which incidentally strips both the OS-illegal filename set `\ / : * ? " < > |` and A#6's forbidden link-control chars `# | [ ] ^`), collapse repeated `-`, trim leading/trailing `-`, cap at ~200 bytes. **Keep Unicode letters verbatim, no transliteration** — Obsidian does none, so `café → café` is the compatible choice. Lowercasing is load-bearing: it sidesteps the unimplemented case-insensitive-alias gap (`resolveEdges` uses byte-exact `=`; no `COLLATE NOCASE`) for every tool-minted slug, so I do **not** add `COLLATE NOCASE` in this ticket (a T1/T2-territory change; flagged as a known limitation for hand-authored mixed-case aliases). *Alternative considered:* import legacy `internal/parser.NormalizeSlug` (functionally near-identical) — rejected on boundary hygiene (`internal/node/AGENTS.md` frames `internal/parser` as legacy-only).

### Where the slugify helper lives
`internal/cli/slug.go`. *Alternatives:* (a) `internal/parser.NormalizeSlug` — crosses the legacy/v1 boundary; (b) promote into `internal/node` — rejected because slug policy is a *tool convention* (like todo's repeater parsing, which lives in `cli`), not a node-format concern; the node package stays format-only. YAGNI: promote later if a second tool needs it.

### Alias-collision behavior at create and rename
**Reject at write time via a filesystem scan of `notes/**/*.md`** (the note-vs-note collision — EC-1/EC-2 — is the case that matters and is a hard `os.Stat` failure on the target filename anyway, matching `addDurableTodo`'s no-clobber precedent). The scan also checks each note's parsed `aliases:` for the candidate slug (catches a slug claimed as an alias on a differently-named note). The **truly global, cross-type** surface (a note slug colliding with a *todo's* alias — EC-14) stays the existing reactive reconcile `alias_collision` warning: that is what "index-enforced uniqueness" means in shipped code, and honoring the design's stronger "rejected" prose for the note-vs-note case without adopting an index dependency at create time avoids two footguns — the index-staleness race (a very recent un-reconciled write from another device is invisible) and introducing a new "an add-verb touches the index" pattern (no existing create path does). Provide a `--slug` override so a user can keep a colliding *title* while choosing a distinct filename. *Alternatives:* (a) index-query-before-write — rejected (staleness + new pattern); (b) auto-disambiguate to `slug-2` — rejected: silently renaming a browse-heavy note file is more surprising than an error; (c) accept the warning-only contract for *all* cases — rejected as too weak for the common duplicate-title case.

### Rename command surface
**A dedicated `rk note rename <ref> "New Title"` verb.** Clearest, matches the verb-per-operation family (`todo done`, `todo add`), and AC-6 needs *only* rename (general `rk note edit` is out of scope per the AC doc). *Alternative:* `rk note edit --title` — rejected as importing an editing surface not required by "Done when". Flow: resolve `ref` via a `notes/**/*.md` glob (filename slug / alias / ULID), CRLF-guard the read; compute `oldSlug` (current filename base) and `newSlug = slugify(newTitle)` (validate non-empty/non-`index`/non-colliding); update the title prop (`SetField` if `HasField("title")`, else `InsertField`); `SetAliases(dedupe(existing ∪ {oldSlug, newSlug}))` so both the retained old slug and the new canonical slug resolve; `writeFileAtomic` to the new path in the same subdirectory, then `os.Remove` the old path (a `git mv` equivalent, so Obsidian never sees two files for one ULID). `id:` never changes (Invariant 2). If `newSlug == oldSlug`, update the title in place with no file move. No other file is touched (zero link churn).

### Block-list `aliases:` handling (the architecturally significant gap)
**Add `internal/node.(*Node).SetAliases([]string)` backed by new `blockSpans` tracking**, gated by the existing round-trip fuzz corpus plus a targeted test. It collapses a block-list `aliases:` to canonical flow form on the (already-mutating) rename path, which is the safe, correct resolution. *Alternatives:* (b) normalize aliases to flow form via ad-hoc CLI text surgery — rejected (duplicates frontmatter-scanning logic outside the node package, the exact coupling `internal/node` exists to prevent); (c) fix `InsertField` to *error* on a block-list key and have `rk note rename` refuse such notes with a helpful message — this is the documented-limitation fallback (turns silent corruption into a clean refusal) and is acceptable only if the `SetAliases` primitive is deemed too much scope, but it leaves AC-6 unmet for Obsidian-authored vaults, so it is the fallback, not the recommendation.

### index.md generation choice
**A `rk note index` porcelain, not folded into `Reconcile`.** Reconcile is the lazy read-path run on every `rk todo list`/`rk note show`; emitting files into the *synced* vault as a side effect of a read would be astonishing and would dirty git/Syncthing on every query. An explicit deterministic verb matches the `rk index`/T4/T5 precedent. It opens the index, reconciles, queries `type='note'` nodes + their `title`/`description` props, groups by the note's directory, and writes one `notes/<dir>/index.md` per populated directory (entries sorted by slug for determinism), using **plain markdown links `[title](slug.md)` — not `[[wikilinks]]`** — so the generated catalog does not inject synthetic `references` edges into the graph. Generated `index.md` files are skipped as note candidates by filename and never listed as entries. *Alternative:* fold into reconcile (the AC doc's tentative default, to avoid a third freshness contract) — rejected per the read-path/side-effect reasoning above; the "explicit regeneration" contract is the same one `rk index` already establishes.

### Show output format per CLI conventions
`noteShowResult` struct with `json` tags and a `Pretty()` method, emitted through `output.New(cmd.OutOrStdout(), mode).Print(res)` with `mode, _ := output.ModeFromFlags(jsonFlag, ndjsonFlag)` and the `!(mode==Pretty && quietFlag)` suppression idiom. Fields: `id`, `type`, `title`, `description`, `stage`, `aliases`, `path`, `forward_links []{rel,dst,dst_key}` (from `SELECT rel,dst,dst_key FROM edges WHERE src=?`), `backlinks []{src,rel}` (from `SELECT src,rel FROM edges WHERE dst_key=?`). Resolution query joins `nodes` with `aliases` (`ulid=? OR EXISTS(SELECT 1 FROM aliases WHERE alias=? AND id=nodes.id)`), mirroring `runTodoListE`'s index-open→reconcile→query shape. Backlinks *display* inside `show` is fine (index-derived, satisfies the ticket's named `show` verb); the dedicated `rk note backlinks` verb remains `reckon-23z`, out of scope.

### Create field conventions
`NewNode("note", author, body)`; `--type` overrides the default `note` (free taxonomy governed by the out-of-scope schema file). `Time = time.Now().UTC().Format(time.RFC3339)` — **UTC `Z` form, following `add.go`/`todo.go`**, not the proposal example's illustrative `-04:00` offset (justified: consistency + unambiguity). Props: `title` (from the positional arg, required — empty/invalid-slug rejected), `description` (from `--description`, optional — soft "mandatory-by-convention", omitted if absent, so TS-9 passes), `tags` (from `--tag`, stored as the literal `[a, b]` string — not a parsed list at the node layer, a pre-existing generic behavior), `stage` (from `--stage`, validated against `seedling|budding|evergreen` at the CLI layer — EC-5 — omitted if absent). `Aliases = [slug]` (self-minted, plus any `--alias`). No `updated`/`timestamp` field is ever written (AC-13). `Render`'s canonical key order emits `id,type,time,author,aliases` then props alphabetically (`description,stage,tags,title`) — cosmetically unlike the proposal's illustrative example, but tests assert on parsed fields, not literal key order (per AC-doc §2.7). Body comes from `--body` (optional; wikilinks in it become edges automatically); empty body is legal and round-trips.

### Subdirectory placement
Default flat `notes/<slug>.md`; optional `--dir <subpath>` places the note at `notes/<dir>/<slug>.md`. Slug uniqueness stays vault-wide (global flat namespace) regardless of directory. `rk note index` emits one `index.md` per directory that contains notes.

## Known risks or ambiguities

- **`SetAliases` is the one new node-package primitive and must clear the round-trip fuzz gate.** Collapsing a block list to flow form is a deliberate, localized rewrite of a single field on an explicit mutation (rename), not a general re-render — this is safe, but the `blockSpans` addition to `parseFrontmatter` touches the most invariant-sensitive function in the codebase; every change is checked against `TestRoundTripIdentity`/`FuzzRoundTripIdentity` before anything else. If reviewers prefer zero node-package change, the fallback (fix `InsertField` to error on block-list + `rk note rename` refuses such notes) ships AC-6 as a documented limitation for Obsidian-block-list vaults.
- **Case-insensitive alias matching remains unimplemented** (`resolveEdges` uses byte-exact `=`, no `COLLATE NOCASE`). Sidestepped for tool-minted slugs by consistent lowercasing; a hand-authored mixed-case `aliases:` entry won't resolve a differently-cased link. Pre-existing T1/T2 gap, flagged as a known limitation rather than fixed here (fixing it is an index-schema change out of T8's scope).
- **Cross-type global collisions (note slug vs. todo alias, EC-14) are surfaced reactively, not rejected at create.** This matches shipped `alias_collision` semantics; only note-vs-note is rejected proactively. If stronger global enforcement is later required, it belongs in a dedicated index-enforcement change, not T8.
- **`Render`'s frontmatter key order won't match the proposal's illustrative example** (props sort alphabetically, so `title` renders after `tags`). Cosmetic only; tests assert on parsed fields. Promoting `title`/`description` to reserved keys for nicer ordering is a node-package change with broad blast radius — deliberately not done.
- **Generated `index.md` becomes an indexed node** (bodyless-frontmatter, surrogate `file:` key). Neutralized by using markdown links (not wikilinks) so no synthetic edges are injected, and by skipping `index.md` as a note candidate. Low-risk but worth a regression assertion.
- **`--dir` subdirectory placement is a plan decision, not a ticket-specified mechanism** (the proposal's tree diagram shows `notes/<topic>/` but never says how `<topic>` is chosen). Flat-default + optional `--dir` is the minimal, non-surprising choice; if a derived-from-tags convention is later wanted, it layers on without rework.
- **Legacy `rk note`/`rk notes` coexistence.** New subcommands attach to the existing `noteCmd` and rely on the per-subcommand `requiresDB:"false"` annotation to skip legacy DB init; the legacy `new`/`list` subcommands keep their DB path. Full consolidation is `reckon-29ln`, explicitly out of scope.
